Decisiones de diseño

¿Por qué Verdi Flow como API para los SDKs?
Corre dentro de Meli — acceso interno a repos SDK sin exponer credenciales afuera
GitHub Actions es externo. Centralizar en Verdi evita gestionar tokens de los 7 SDK repos como secrets.
¿Por qué fury_openapi como intermediario?
Formato OpenAPI 3.1 estándar + capa editorial del equipo DevDocs
Spec Hub devuelve 3.0 sin review. fury_openapi permite review humano, overlays por país y schemas curados.
¿Cuándo entra Verdi en el flujo?
Solo al final — cuando el openapi ya está validado, con schemas y categorizado por país
Verdi no obtiene specs de Spec Hub. Recibe el webhook DESPUÉS de que bundle.py procesa todo y distribuye el spec ya listo a los SDK repos.
